# React Flow 源码导读文章证据索引

## 写作设定

- 输出语言：简体中文。
- 交付模式：`Article - Deep Dive`。新版 roadmap 的主线是先读懂 React Flow / xyflow 源码，mini-flow 只作为后半部分验证练习。
- 受众：有 React 和 TypeScript 基础、想系统读懂 React Flow 源码的前端工程师。
- 核心问题：React Flow 如何把 graph data、viewport、renderer、interaction-to-change pipeline 和 React API 组织成一个图编辑器运行时。
- 贯穿例子：先用 React Flow public API 建立问题域，后续再用 `mini-flow` 复刻关键机制。
- 本地 wiki：当前仓库未发现 `wiki/` 目录，因此本轮按 `roadmap.md` 与 xyflow 本地 research context 写作。
- 参考样文：`learn-agent/src/content/blog/zh/AI/3.ClaudeCode源码解析`。本轮已吸收其“先破误解、再建运行时图景、再给承重链路和源码坐标”的写法。

## 风格契约

- 开篇从常见误解切入，不直接堆定义。
- 每篇要有一个明确的工程问题，而不是文件清单。
- 用“承重链路”组织源码入口，少按目录树平铺。
- 关键概念要解释为什么出现、解决什么问题、带来什么取舍。
- 源码证据要落到具体路径，重要判断不只凭概括。
- 前 18 篇以源码导读为主，代码只作为最小草图，避免提前变成造轮子教程。
